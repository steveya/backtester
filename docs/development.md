# Development

## Local setup

Use the development extras for local work:

```bash
pip install -e ".[dev]"
```

This installs the package plus the current development tools, including test,
lint, and documentation dependencies.

## Common commands

Run the test suite:

```bash
python -m pytest tests/ -v
```

Run lint checks:

```bash
ruff check backtester/ tests/
ruff format --check backtester/ tests/
```

Build the documentation site:

```bash
mkdocs build --strict
```

Run the synthetic execution-clock example:

```bash
python examples/execution_walkforward_synthetic.py
```

## Documentation workflow

Documentation source files live in `docs/`, and the site configuration lives in
`mkdocs.yml`.

BAC-1 standardizes on:

- MkDocs for the documentation site
- curated Markdown pages rather than autogenerated API docs
- strict CI builds so docs errors fail pull requests

When editing docs:

- keep the content aligned with the code that actually ships
- prefer stable internal links over hard-coded external references
- document planned work only as future work, not as current functionality

When adding examples:

- keep them provider-agnostic
- use synthetic or preconstructed local data
- show the application/library boundary explicitly rather than embedding loader logic

## CI checks

The GitHub Actions workflow runs:

- lint checks
- test runs
- a dedicated docs build job using `mkdocs build --strict`

The docs job is build-only for now. It validates the site configuration and
internal documentation links but does not publish a site artifact.
